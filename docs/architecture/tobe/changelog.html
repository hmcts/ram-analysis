v2.11 — Retire the RFC process; post-verification sitting amendments handled via UI re-open + RBAC 2026-05-11 Process change: the as-is APEX "Request for Change (RFC)" external workflow for amending verified sittings is retired in RAM Pathfinder. Post-verification amendments are handled inside the UI by a re-open action gated by RBAC, with mandatory in-form justification and full audit. RAM Pathfinder's first-class RBAC model (FR2) and per-request authz/check make that ticketing unnecessary — the same gate is achievable inside the UI with a permission, a justification field, and an audit row. Reduces friction and brings amendment handling into the same trace as every other write.
